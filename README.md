# Combined bis draft for RFC5884 and RFC 7726


The expected work:
- Generate a -bis document as an I-D. I would suggest that this be done in the IETF's github instance to help with other tracking such as issues and storing of intermediate work artifacts.  The IETF wiki is a good place to consider keeping the conformance reports that will come out of this effort.
- Incorporate known errata into it.  As noted in-thread, some of this is already done by the tooling! 
- Review the drafts vs. implementations.  It's my belief at this point that the features covered by RFCs 5880-5884 are fully deployed.  A formal survey to confirm this would be helpful.  This perhaps turns into a full conformance report - Ketan will likely have some advice as to what the IESG is likely to want to see.  Note that RFC 6410 removed the requirement for a full interoperability report.
- Work through the RFC 2026/6410 process.  As you'll note, there are a number of RFCs extending the process.  Additionally, the procon WG is doing work that also addresses the standards process.  The IESG will likely have guidance for how they want this process to move forward.

One item that will require some additional chasing, primarily overlapping RFC 5884-bis, is original author acknowledgment/contact.  For the other RFCs, the Dave's are still in the industry, although notoriously hard to reach.  I can contact Dave Katz fairly easily, Dave Ward usually takes a bit more work.  Tom Nadeau is less active but still reachable. I can manage Kireeti. George has, I believe, retired.  Rahul is inactive in IETF to the best of my knowledge, but may still be reachable.  I suspect the collective social network for the people in this thread will be able to reach everyone.
